# Persona: The Product Founder

> "I have a vision for what to build — I need AI to help me figure out *what matters most* and then actually build it without a 10-person team."

## Profile

| Field | Value |
|-------|-------|
| **Name** | "Sara" — The Product Founder |
| **Age** | 30-50 |
| **Role** | Product manager, technical founder, or solo entrepreneur |
| **Team size** | 0-5 (often just themselves + AI) |
| **Experience** | Strong product sense, variable coding ability (from none to intermediate) |
| **Tools today** | Productboard, Linear, Notion, Figma, ChatGPT/Claude for brainstorming |
| **Spending** | $50-500/month on SaaS tools; ROI-driven, will pay for leverage |
| **Tech comfort** | Medium — can use CLI if docs are clear, prefers structured workflows over raw code |

## Behaviors

- Thinks in terms of user problems, not technical solutions
- Maintains a mental backlog of 50+ feature ideas but struggles to prioritize objectively
- Reads customer feedback obsessively — support tickets, reviews, social media
- Gets frustrated when developers build the wrong thing because requirements were unclear
- Values frameworks (Jobs-to-be-Done, VPC, Lean Canvas) for making product decisions
- Wants to move fast but worries about shipping half-baked features
- Evaluates tools by asking "does this help me ship the *right* thing faster?"

## Value Proposition Canvas

### Customer Jobs

| Type | Job |
|------|-----|
| Functional | Prioritize feature backlog based on real user needs, not gut feeling |
| Functional | Translate product vision into structured requirements developers can execute |
| Functional | Validate feature ideas against user personas before investing dev time |
| Functional | Ship features end-to-end without managing a large engineering team |
| Social | Demonstrate to investors/stakeholders that product decisions are data-driven |
| Emotional | Feel confident that what's being built actually matters to users |
| Emotional | Reduce anxiety about building the wrong thing |

### Pains

| Severity | Pain |
|----------|------|
| Critical | No structured way to connect product discovery to engineering execution |
| Critical | Feature ideas pile up with no objective way to rank them against user needs |
| High | AI coding tools can build *anything* but don't help decide *what* to build |
| High | Translating product requirements into technical tasks is a bottleneck |
| High | Competitive landscape moves fast — hard to stay current on what users actually want |
| Medium | Persona creation is manual, time-consuming, and often based on assumptions not research |
| Medium | Backlog grooming sessions feel unproductive without a scoring framework |
| Low | Existing AI PM tools (Productboard AI, Jira AI) are siloed from the development workflow |

### Gains

| Impact | Gain |
|--------|------|
| High | VPC-scored backlog that objectively ranks features by persona impact |
| High | End-to-end pipeline: discover → prioritize → architect → implement → ship |
| High | AI-generated personas based on competitive research, not guesswork |
| Medium | Automatic feature ideation grounded in actual user pain points |
| Medium | See exactly what the AI architect designed before implementation starts |
| Medium | Reduce dependency on large engineering teams for feature delivery |
| Low | Backlog that syncs with GitHub Issues or JIRA — no separate tool to maintain |
| Low | Product discovery that runs continuously, not just during quarterly planning |

## Key Insight

> Current AI tools create a **product-engineering gap**: PM tools help discover *what* to build, coding tools help build *how*, but nothing connects them. The founder who can describe their users and let a pipeline handle discovery → architecture → implementation → review → ship has a massive leverage advantage over those manually bridging each step.
