## ADDED Requirements

### Requirement: sr-architect scaffolds artifacts via opsx CLI
The sr-architect agent SHALL invoke `/opsx:new <specName>` followed by `/opsx:ff <specName>` as its mandatory first step before producing any design output. The agent SHALL NOT hand-author proposal.md, design.md, or tasks.md directly.

#### Scenario: Architect scaffolds artifacts on invocation
- **WHEN** sr-architect is launched with a `specName` argument
- **THEN** it first calls `/opsx:new <specName>` to register the change directory
- **THEN** it calls `/opsx:ff <specName>` to generate all artifacts (proposal.md, design.md, tasks.md, context-bundle.md)
- **THEN** it proceeds with analysis using the generated artifacts as the foundation

#### Scenario: Architect halts when specName is absent
- **WHEN** sr-architect is launched without a `specName` argument
- **THEN** it halts immediately and reports: "specName is required — invoke this agent with the change name as argument"

#### Scenario: Architect does not hand-author artifacts
- **WHEN** sr-architect completes its work
- **THEN** proposal.md, design.md, and tasks.md exist in `openspec/changes/<specName>/` as generated by the opsx CLI, not written by the agent directly

### Requirement: sr-architect self-trigger is removed
The sr-architect frontmatter `description` field SHALL NOT contain language that triggers the agent when a user types `/opsx:ff`. The agent SHALL be launched by an orchestrator or user with an explicit `<specName>` argument.

#### Scenario: No circular invocation
- **WHEN** a user invokes `/opsx:ff`
- **THEN** the `/opsx:ff` slash command executes its own logic — it does NOT re-launch sr-architect as a nested sub-agent
- **THEN** if sr-architect is needed, the orchestrator launches it with a specName argument

### Requirement: sr-developer implements via opsx:apply
The sr-developer agent SHALL invoke `/opsx:apply <specName>` at the start of Phase 3 rather than writing files directly to the repository. The `specName` argument is the spec name passed as the agent's invocation argument.

#### Scenario: Developer invokes apply before writing files
- **WHEN** sr-developer enters Phase 3
- **THEN** it calls `/opsx:apply <specName>` as the first action
- **THEN** it does not write production files to the repo outside of the /opsx:apply workflow

#### Scenario: Developer halts when specName is absent
- **WHEN** sr-developer is launched without a `specName` argument
- **THEN** it halts immediately and reports: "specName is required — invoke this agent with the change name as argument"

### Requirement: sr-developer verifies checkbox completion before handoff
The sr-developer agent SHALL read `openspec/changes/<specName>/tasks.md` after `/opsx:apply` exits and verify that all checkboxes show `[x]`. If any checkbox matches `- [ ]`, the agent SHALL NOT proceed to Phase 4 and SHALL report the incomplete tasks.

#### Scenario: All tasks complete — handoff proceeds
- **WHEN** `/opsx:apply <specName>` exits
- **AND** all lines in tasks.md matching checkbox syntax show `[x]`
- **THEN** the developer proceeds to Phase 4 (Verify)

#### Scenario: Incomplete tasks block handoff
- **WHEN** `/opsx:apply <specName>` exits
- **AND** tasks.md contains one or more lines matching `- [ ]`
- **THEN** the developer HALTS and does not proceed to Phase 4
- **THEN** it reports each incomplete task by title

### Requirement: sr-reviewer gates archive on task completion
The sr-reviewer agent SHALL read `openspec/changes/<specName>/tasks.md` and assert that all checkboxes show `[x]` before invoking `/opsx:archive <specName>`. If any checkbox matches `- [ ]`, the agent SHALL block archive and report the incomplete tasks.

#### Scenario: All tasks complete — archive proceeds
- **WHEN** sr-reviewer completes CI checks and all checks pass
- **AND** all lines in `openspec/changes/<specName>/tasks.md` matching checkbox syntax show `[x]`
- **THEN** the reviewer invokes `/opsx:archive <specName>`

#### Scenario: Incomplete tasks block archive
- **WHEN** sr-reviewer reads tasks.md
- **AND** any line matches `- [ ]`
- **THEN** the reviewer blocks archive and reports: "Archive blocked: the following tasks remain incomplete: [list of task titles]"
- **THEN** `/opsx:archive` is NOT invoked

#### Scenario: Reviewer halts when specName is absent
- **WHEN** sr-reviewer is launched without a `specName` argument
- **THEN** it halts immediately and reports: "specName is required — invoke this agent with the change name as argument"

### Requirement: specName is a required positional argument for all three core agents
All three agents (sr-architect, sr-developer, sr-reviewer) SHALL treat the first invocation argument as `specName`. The argument is required; no fallback or derivation is permitted.

#### Scenario: specName forwarded to every opsx command
- **WHEN** an agent is launched with `specName = "my-feature"`
- **THEN** every opsx command invoked by that agent uses `"my-feature"` as the positional argument
- **THEN** the specName is never hardcoded or omitted from any opsx invocation
